                                          Al Al • a month ago

                                          Figured it out! I'm using Rails 3.2.3 which doesn't use strong_parameters. So my params were a hash, rather than a Ruby Object. 3.2 uses attr_accessible in the Post model, where I put this:
                                          attr_accessible :caption, :image

                                          and then in the post_params method I just accessed the params like a regular hash:

                                          def post_params
                                          params[:post]
                                          end

                                          This probably isn't the best way to do this, but it's working. Any feedback would be greatly appreciated. Thanks again for this tutorial!

                                                                                  Ben Walker Mod Arjun Menon • 2 months ago

                                                                                  Hi Arjun. You shouldn't delete migrations, because it doesn't actually affect the database in anyway and you lose that record for the sake of rollbacks. If you think the app is beyond repair, clone my final commit for Part 1: https://github.com/benwalks/ph... and try again.

                                                                                  Before that though, try this in your rails console.

                                                                                  user = User.first
                                                                                  user.posts

                                                                                  If user.posts returns an empty record, you're on the right track. If not, your models must still be a bit out of whack. The "User.post" will give you a no method error, because User is the whole model. Think of it this way:

                                                                                  User.all # Will return you all of the users in the User model.
                                                                                  Post.all #Will return you all of the posts in the Post model
                                                                                  User.post #Doesn't mean anything.

                                                                                  So by assigning User.first or User.last or whatever, to a variable user and then running the posts method afterwards you'll be at the right 'level' to access an individual ussers posts or an empty object if none exist.

                                                                                  The id thing is fine. As Users are created the id will iterate one by one, the same goes for Posts. They should only be linked in that the user_id of an individual post should be linked to the id of a User.

                                                                                  I just saw your update!

                                                                                  And unfortunately I'm not sure why that would be the case. What's the alternative that works? My new article is just being proof-read and edited so you won't have long to wait to see how I've added users.
